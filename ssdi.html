SDLC Cheat Sheet

Software Development Life Cycle (SDLC)

The SDLC is a framework that defines the steps involved in the development of software. Here's a comprehensive cheat sheet for each phase:

1. Requirements Gathering

  • Identify stakeholders and end-users
  • Conduct interviews and surveys
  • Create user stories and use cases
  • Define functional requirements
  • Define non-functional requirements
  • Validate requirements with stakeholders
  • Create Software Requirements Specification (SRS)
  • Establish prioritization for requirements
üîç

2. System Analysis

  • Analyze existing systems (if applicable)
  • Create data flow diagrams
  • Develop entity-relationship diagrams
  • Identify system boundaries
  • Define integration points
  • Analyze technical feasibility
  • Conduct risk assessment
  • Evaluate alternative solutions

3. System Design

  • Create high-level architecture design
  • Design database schemas
  • Define module specifications
  • Create class diagrams
  • Design API specifications
  • Create UI/UX wireframes and mockups
  • Define security architecture
  • Document design decisions
üíª

4. Implementation

  • Set up development environment
  • Create coding standards
  • Implement database
  • Develop core functionalities
  • Implement UI components
  • Write unit tests
  • Conduct code reviews
  • Perform continuous integration

5. Testing

  • Create test plans and test cases
  • Perform unit testing
  • Conduct integration testing
  • Execute system testing
  • Perform user acceptance testing (UAT)
  • Test performance and load handling
  • Conduct security testing
  • Document and track defects

6. Deployment

  • Create deployment plan
  • Prepare production environment
  • Create user documentation
  • Train end-users
  • Migrate data (if applicable)
  • Deploy application to production
  • Conduct post-deployment testing
  • Get stakeholder sign-off

7. Maintenance & Support

  • Monitor system performance
  • Address bug fixes
  • Implement feature enhancements
  • Apply security patches
  • Optimize performance
  • Conduct regular backups
  • Provide technical support
  • Plan for system upgrades

SDLC Models Comparison

Waterfall Model

  • Sequential approach
  • Each phase must be completed before next begins
  • Well-documented process
  • Easy to understand and manage
  • Best for small projects with clear requirements
  • Difficult to accommodate changes
  • Testing starts after development is complete
üîÑ

Agile Model

  • Iterative and incremental approach
  • Divides project into small iterations (sprints)
  • Embraces changing requirements
  • Continuous customer involvement
  • Regular delivery of working software
  • Daily stand-up meetings
  • Retrospectives to improve process
üèâ

Scrum Framework

  • Popular Agile implementation
  • 2-4 week sprint cycles
  • Uses product backlog and sprint backlog
  • Defined roles: Product Owner, Scrum Master, Team
  • Daily stand-up meetings
  • Sprint review and retrospective
  • Burn-down charts for tracking progress

DevOps Model

  • Integration of development and operations
  • Continuous integration and delivery
  • Automated testing and deployment
  • Infrastructure as code
  • Continuous monitoring
  • Rapid feedback loops
  • Focus on collaboration and communication
